# 2026-06-06 更改日志

## 1. 页面切换后翻译状态恢复

**问题**：翻译进行中切换到其他页面再跳回来，所有翻译信息和进度丢失。

**改动**：
- `server/server.py` — 新增 `GET /api/task/{task_id}` 端点，返回任务当前状态 JSON
- `server/translate.html` — localStorage 持久化 `taskId` + 文件元数据；页面加载时自动检测并恢复（重连 SSE 进度条）
- `server/batch.html` — localStorage 持久化所有批量任务；页面加载时批量恢复，运行中的自动重连 SSE

**恢复逻辑**：
- 翻译进行中 → 重连 SSE 流，进度条恢复显示
- 翻译完成 → 显示下载按钮
- 翻译出错/停止 → 显示重试按钮
- 超过 24 小时 → 自动清理过期任务
- 上传新文件 → 自动清理旧任务状态

## 7. PDF 指定页码翻译

**三个引擎均支持页码选择**：

| 引擎 | 参数格式 | 示例 |
|------|----------|------|
| BabelDOC | `TranslationConfig(pages=str)` | `"1-5,8,10-12"` |
| PDFMathTranslate | `--pages PAGES` | `"1-5,8,10-12"` |
| Native PyMuPDF | extract 阶段过滤 | `"1-5,8,10-12"` |

**改动**：
- `handlers/pdf_handler.py` — `rebuild_via_pdf2zh()` + `extract()` 增加 `pages` 参数
- `handlers/pdf_babeldoc_handler.py` — `translate_full()` + `translate_full_async()` 增加 `pages` 参数
- `server/server.py` — start 端点从请求体解析 `pages` 并传递给各引擎
- `server/translate.html` — PDF 上传后引擎选择器旁显示页码输入框
- `server/batch.html` — 控制栏增加全局页码输入框
