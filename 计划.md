# EpubTranslator 项目总结与后续计划

> 2026-06-02 | 当前 commit: `1701a67`

## 一、项目现状

### 1.1 已实现的翻译能力

| 格式 | 引擎 | 状态 |
|------|------|------|
| EPUB | 自研 (BeautifulSoup + DeepSeek) | ✅ 可用，双语/纯中文，HTML 块级翻译 |
| DOCX | 自研 (python-docx) | ✅ 可用 |
| PDF | PDFMathTranslate (pdf2zh) | ✅ 可用，排版好，默认引擎 |
| PDF | 自研原生 (PyMuPDF exact bbox) | ⚠️ 备选，字号可能偏小 |

### 1.2 Web 服务

- FastAPI + SSE 实时进度
- 上传 → 翻译 → 下载 全流程
- 设置面板：API Key / 模型 / 翻译模式
- 启动脚本：`启动.bat` (Windows) / `启动.sh` (Linux)
- PDF 自动走 pdf2zh，EPUB/DOCX 走自研

### 1.3 关键依赖

```
# 自研
beautifulsoup4, lxml, openai, pyyaml, tqdm, cryptography, langdetect
fastapi, uvicorn, python-multipart, python-docx, PyMuPDF

# PDF增强 (需 Python 3.10-3.12)
pdf2zh (PDFMathTranslate, 通过 uv 安装)
```

## 二、PDF 翻译探索历程

### 2.1 走过的路

| 阶段 | 方案 | 结果 |
|------|------|------|
| 1 | 原地替换，逐行写回 | 碎片化翻译，中文溢出 |
| 2 | 段落合并提取 | 上下文改善，但 PDF block 分界不准 |
| 3 | exact bbox 缩字 | 布局不乱，长段落字号极小 |
| 4 | 流式排版 | 位置混乱 |
| 5 | 列感知流式 | 仍然混乱 |
| 6 | 集成 PDFMathTranslate | **当前最优解**，排版好但跳过小字/公式区域 |
| 7 | pdf2zh + 原生混用 | 排版不一致，**已回退** |

### 2.2 当前 PDF 方案

- **默认**：pdf2zh（PDFMathTranslate）—— pdfminer 内容流级重建
- **参数**：`--vfont "" --vchar "" --ignore-cache`（最大化翻译范围）
- **字体**：SourceHanSerifCN（思源宋体，原生和 pdf2zh 统一）
- **限制**：≤6pt 小字、AI 判定为"非正文"的区域不翻译

### 2.3 已放弃的方向

- ❌ 混用两个引擎（排版不一致）
- ❌ babeldoc 实验后端（NumPy 不兼容）
- ❌ 流式/列感知排版（位置准确性不可控）

## 三、PDF DOM 研究计划

### 3.1 目标

自研一个**内容流级别**的 PDF 翻译引擎，替代 pdf2zh，实现对翻译范围和布局的完全控制。

### 3.2 技术路线

```
PDF 文件
  ↓ pdfminer.six 解析
内容流操作符 (Tj, TJ, Tm, Tf...)
  ↓ 在 render_string 回调中拦截文字
翻译英→中
  ↓ 调整 Tm 矩阵 + 嵌入 CJK 字体
生成新的内容流
  ↓ PyMuPDF / pikepdf 组装
输出 PDF
```

### 3.3 参考项目

- **PDFMathTranslate** ([GitHub](https://github.com/Byaidu/PDFMathTranslate))：pdfminer.six + 自定义 PDF 操作符生成
- **pdfminer.six** ([GitHub](https://github.com/pdfminer/pdfminer.six))：PDF 内容流解析库
- **pikepdf**：PDF 底层编辑（基于 QPDF）

### 3.4 分阶段研究

| 阶段 | 内容 | 预计时间 |
|------|------|---------|
| 1 | 读懂 PDF 内容流操作符 | 1-2 天 |
| 2 | 理解 pdfminer.six 解析架构 | 3-5 天 |
| 3 | 实现文字写回（render_string → 新操作符） | 5-10 天 |
| 4 | 打磨（旋转、多栏、表格、图形） | 持续 |

详细提纲见 Obsidian：`OCP-Project/DingTranslator/PDF-DOM研究提纲.md`

## 四、关键文件索引

## 五、新任电脑设置

```bash
git clone <repo>
cd EpubTranslator
pip install -r requirements.txt
uv tool install --python 3.12 pdf2zh    # PDFMathTranslate
python 启动.sh                           # 或 Windows: 启动.bat
```

然后在 `config.yaml` 中填入 DeepSeek API Key。
